# Postman

## Обзор

Этот проект моделирует систему доставки писем почтальонами.
В системе несколько почтальонов получают письма на день, доставляют их и получают зарплату в зависимости от количества писем, условий работы и результатов доставки.

Цель проекта — смоделировать доменную логику распределения писем, расчёта зарплаты, бонусов и штрафов.

---

# Основная идея

В начале дня система содержит структуру данных, которая показывает, сколько писем должен доставить каждый почтальон.

Пример:

```go
Map<PostmanId, LettersAssignedToday>
```

Пример данных:

```go
{
  1: 120,
  2: 95,
  3: 140
}
```

Где:

- **ключ** — ID почтальона
- **значение** — количество писем, назначенных на сегодня

---

# Расчёт зарплаты

Зарплата почтальона рассчитывается по формуле:

```go
salary = lettersAssigned × baseRate × dayCoefficient × weatherCoefficient × experienceCoefficient
```

## Параметры

**lettersAssigned**
Количество писем, которые почтальон должен доставить.

**baseRate**
Базовая ставка за одно письмо.

### Коэффициент дня

| Тип дня  | Коэффициент |
| -------- | ----------- |
| Будний   | 1.0         |
| Выходной | 1.5         |

### Коэффициент погоды

| Погода     | Коэффициент |
| ---------- | ----------- |
| Нормальная | 1.0         |
| Дождь      | 1.2         |
| Снег       | 1.3         |
| Шторм      | 1.5         |

### Коэффициент опыта

| Уровень | Коэффициент |
| ------- | ----------- |
| Junior  | 0.9         |
| Middle  | 1.0         |
| Senior  | 1.2         |

---

# Ограничения на количество писем

У каждого почтальона может быть максимальное количество писем, которое он способен доставить за день.

| Уровень | Максимум писем |
| ------- | -------------- |
| Junior  | 80             |
| Обычный | 120            |
| Senior  | 160            |

Если почтальону назначено больше писем, чем он может доставить:

- лишние письма могут перераспределяться другим почтальонам
- либо переноситься на следующий день

---

# Бонусы

Если почтальон доставил больше писем, чем было запланировано, он может получить бонус.

Формула:

```go
bonus = extraLetters × bonusRate
```

Пример:

```go
bonusRate = baseRate × 0.5
```

---

# Штрафы

Если почтальон не доставил все письма, может применяться штраф.

Пример формулы:

```go
penalty = undeliveredLetters × baseRate × penaltyCoefficient
```

Альтернативный вариант:

```go
penalty = sqrt(undeliveredLetters) × baseRate
```

---

# Причины недоставки писем

Не все недоставленные письма должны приводить к штрафу.

Возможные причины:

```bash
CLIENT_NOT_HOME // + если у него нету почтового ящика
WRONG_ADDRESS
BAD_WEATHER
POSTMAN_FAULT
```

Штраф применяется только если причина:

```go
POSTMAN_FAULT
```

# Типы писем

Разные типы писем могут иметь разную оплату.

| Тип        | Множитель |
| ---------- | --------- |
| SIMPLE     | 1.0       |
| REGISTERED | 1.5       |
| EXPRESS    | 2.0       |

Формула:

```go
payment = baseRate × typeMultiplier
```

---

# Длина маршрута

У маршрута может быть длина, которая влияет на оплату.

Пример:

```go
distanceCoefficient = 1 + (distanceKm / 100)
```

Чем длиннее маршрут, тем выше коэффициент оплаты.

# Статистика почтальона

Система может хранить статистику каждого почтальона.

```bash
PostmanStats
- deliveredTotal
- undeliveredTotal
- rating
```

Рейтинг может влиять на коэффициенты зарплаты.

---

# Отчёт за день

Каждый рабочий день формируется отчёт по доставке.

```bash
PostmanDayReport
- postmanId
- lettersAssigned
- lettersDelivered
- lettersUndelivered
- bonuses
- penalties
- finalSalary
```
